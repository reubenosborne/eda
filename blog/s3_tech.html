A closer look..

HTML & CSS

Hyper Text Markup Language, Cascading Style Sheets, not immediately decipherable from the get go but relatively simple to understand.

Here's how I described the difference to a friend the other day:

Imagine you make a chair. You start with a block of something to sit on. You give it four legs. You want a backrest as it's not a stool. HTML creates these elements of an object (website) with none of the details. It's the raw shape or form of something.

CSS then comes along and transforms the chair (website). It gives the chair size, material, thickness, colour.

Bonus: Javascript gives the chair it's functionality. Does the chair fold closed? Does it stack? Can it be moved?

Control flow

What is control flow?

Control flow is quite simply the order in which code is read, generally, from top to bottom in sequence.

Loops are when code will repeat a certain section of code until a certain criteria is met.

It's comparable to following a recipe, or baking a cake. You follow the instructions step by step but instead of a recipe saying:

      Sift Flour
      Crack egg into bowl
      Crack egg into bowl
      Crack egg into bowl
      Crack egg into bowl
      Melt butter

A loop would denote multiple steps differently:

      Sift Flour
      For each egg you're using, crack it into bowl
      Melt Butter

Loops ultimately save time, have less room for error, are more accepting of change and convey information cleanly and concisely.

The DOM

The DOM, or 'Document Object Model' is a singular object that is created as a sort of map of a website's elements. The DOM assists in Javascript's ability to manipulate HTML & CSS on a website.

The ways you can interact with the DOM are virtually endless. Elements of a website that are visible and presented on the screen can be manipulated, hidden or shown. Their attributes can be altered, giving them different visual characteristics - perhaps to alert an user of something particular like a warning.

Arrays & Objects

Arrays and objects have similarities (Arrays are objects!!) but the situations in which we would use them and the ways in which we interact with them are different.

Arrays are akin to lists. You can store all sorts of things within an array. This 'list' has an 'index', a means of accessing each individual entry. Entries can be removed, altered, added with various commands.

Objects can also store pieces of data but in the form of a 'key/value' pairing. The key esentially being a name or description of the value.

Functions

Functions are here to make our lives far simpler. They're executable snippets of code that.. do things. Things that we tell them to do.

Functions are essentially commands. If a function relies on information it does not start with then we can give the hungry l'il function 'arguments'.

They're able to be reused so they speed up our ability write an application/website. This also makes our code cleaner.
